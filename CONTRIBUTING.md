# Contributing to pytest-ui

We welcome contributions to pytest-ui! Please take a moment to review these guidelines before submitting issues or pull requests.

## How to Contribute

### Reporting Bugs

*   **Check for existing issues:** Before opening a new issue, please search existing issues to see if the bug has already been reported.
*   **Provide detailed information:** When reporting a bug, include:
    *   A clear and concise description of the bug.
    *   Steps to reproduce the behavior.
    *   Expected behavior.
    *   Screenshots or error messages if applicable.
    *   Your operating system, browser, and project version.

### Suggesting Enhancements

*   **Check for existing suggestions:** Search existing issues to see if a similar enhancement has already been proposed.
*   **Describe the enhancement:** Clearly explain the proposed enhancement and why it would be beneficial.
*   **Provide use cases:** Illustrate how the enhancement would be used.

### Submitting Pull Requests

1.  **Fork the repository:** Create a fork of the project to your GitHub account.
2.  **Clone your fork:** Clone the forked repository to your local machine.
3.  **Create a new branch:** Create a new branch for your changes (e.g., `feature/my-new-feature` or `bugfix/fix-issue-123`).
4.  **Make your changes:** Implement your changes and ensure they adhere to the project's coding style (if specified).
5.  **Write clear commit messages:** Use descriptive and concise commit messages.
6.  **Push your branch:** Push your new branch to your forked repository.
7.  **Open a Pull Request:** Open a pull request from your branch to the `main` (or designated default) branch of the original repository.
8.  **Describe your changes:** Provide a clear description of the changes in your pull request, referencing any related issues.
